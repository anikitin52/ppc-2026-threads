## 1. Введение
Численное интегрирование методом Монте-Карло широко применяется для вычисления кратных интегралов в многомерных областях, где классические детерминированные методы становятся вычислительно неэффективными. Основная проблема заключается в том, что последовательная реализация данного алгоритма требует значительного времени при большом количестве точек интегрирования, особенно при увеличении размерности задачи. Ожидаемый результат работы — разработка и сравнение нескольких параллельных реализаций метода Монте-Карло с использованием различных технологий параллелизма (OpenMP, TBB, STL threads) для достижения максимальной производительности при сохранении точности вычислений.

## 2. Формулировка задачи
## 2. Формулировка задачи

Пусть задана непрерывная функция \( f(\mathbf{x}): \mathbb{R}^d \rightarrow \mathbb{R} \), определенная на \( d \)-мерном гиперкубе \( \Omega = [a_1, b_1] \times [a_2, b_2] \times \ldots \times [a_d, b_d] \), где \( a_j, b_j \in \mathbb{R} \), \( a_j < b_j \) для всех \( j = 1, \ldots, d \). Требуется вычислить значение кратного интеграла:

\[
I = \int_{\Omega} f(\mathbf{x}) \, d\mathbf{x} = \int_{a_1}^{b_1} \int_{a_2}^{b_2} \cdots \int_{a_d}^{b_d} f(x_1, x_2, \ldots, x_d) \, dx_d \ldots dx_2 dx_1.
\]

Для приближенного вычисления используется метод Монте-Карло с квазислучайной последовательностью Кронекера:

\[
I \approx V \cdot \frac{1}{N} \sum_{i=0}^{N-1} f(\mathbf{x}_i),
\]

где \( V = \prod_{j=1}^{d} (b_j - a_j) \) — объем области интегрирования, \( N \) — количество точек интегрирования, \( \mathbf{x}_i \in \Omega \) — точки, генерируемые по формуле \( x_{i}^{(j)} = a_j + u_{i}^{(j)} (b_j - a_j) \), а \( u_{i}^{(j)} \in [0, 1) \) — компоненты квазислучайной последовательности Кронекера: \( u_{i}^{(j)} = \{ i \cdot \sqrt{p_j} \} \), где \( p_j \) — \( j \)-е простое число, а \( \{\cdot\} \) обозначает взятие дробной части.

**Входные данные:**
- Вектор нижних границ \( \mathbf{a} = (a_1, a_2, \ldots, a_d) \)
- Вектор верхних границ \( \mathbf{b} = (b_1, b_2, \ldots, b_d) \)
- Целое число \( N > 0 \) — количество точек интегрирования
- Тип тестовой функции \( f \) из заданного набора (константа, линейная, произведение, квадратичная, экспоненциальная)

**Выходные данные:**
- Вещественное число \( I_{\text{approx}} \) — приближенное значение интеграла

**Ограничения:**
- Размерность пространства \( d \geq 1 \)
- \( N \geq 1 \)
- \( a_j < b_j \) для всех \( j \)
- Требуемая относительная погрешность: не более 2-5% в зависимости от типа функции

## 3. Базовый алгоритм (последовательный)
Инициализация
Прочитать входные данные: векторы нижних a и верхних b границ, количество точек N, тип функции f_type.

Вычисление объема области интегрирования
Вычислить V = ∏(b_j - a_j) для всех измерений j = 1..d.

Инициализация суммы
Установить sum = 0.0.

Генерация точек и накопление суммы
Для i = 0 до N-1 выполнить:
4.1 Создать пустой вектор point размерности d.
4.2 Для каждого измерения j = 1..d сгенерировать компоненту точки:

Вычислить alpha = sqrt(prime[j % 10]) (из таблицы простых чисел)

Взять дробную часть alpha

Вычислить u = frac(i * alpha)

Преобразовать в точку области: point[j] = a[j] + u * (b[j] - a[j])
4.3 Вычислить значение функции value = f(point, f_type).
4.4 Добавить value к sum.

Вычисление результата
Вычислить result = V * (sum / N).

Возврат результата
Сохранить result в выходные данные задачи.

## 4. Схема распараллеливания

### 4.1 Декомпозиция

Задача разбивается на `N` независимых подзадач по вычислению значения функции `f(x_i)` в каждой точке. Ключевое свойство метода Монте-Карло — **отсутствие зависимостей по данным** между точками, что позволяет использовать параллелизм по данным (data parallelism).

**Схема декомпозиции для всех реализаций:**
- Исходный диапазон точек `[0, N)` разбивается на непрерывные блоки
- Количество блоков соответствует числу потоков/ядер
- Каждый поток обрабатывает свой блок независимо
- Редукция (суммирование) выполняется после завершения всех вычислений

### 4.2 Планирование

| Технология | Стратегия планирования | Размер блока |
|------------|------------------------|--------------|
| **SEQ** | Отсутствует (один поток) | Весь диапазон `[0, N)` |
| **OpenMP** | Статическое планирование (`schedule(static)`) | `N / num_threads` |
| **TBB** | Рекурсивное разбиение (ворк-стрилинг) | Адаптивный, зависит от загрузки |
| **STL** | Статическое разбиение | `N / num_threads` |

### 4.3 Синхронизация

| Технология | Механизм синхронизации | Редукция |
|------------|------------------------|----------|
| **SEQ** | Не требуется | Последовательное суммирование |
| **OpenMP** | Неявная барьерная синхронизация в конце цикла | `reduction(+:sum)` |
| **TBB** | Автоматическая синхронизация через `parallel_reduce` | Функция объединения `[](double x, double y) { return x + y; }` |
| **STL** | `std::future::get()` — ожидание завершения асинхронных задач | Ручное суммирование результатов из `std::future` |

### 4.4 Особенности ALL версии

Реализация `NikitinAMonteCarloALL` выполняет **автоматический выбор технологии** с приоритетом:

```
TBB → OpenMP → STL → SEQ
```

Выбор осуществляется на этапе выполнения `RunImpl()` через метод `SelectBestTechnology()` с использованием директив препроцессора для проверки доступности библиотек. Это обеспечивает максимальную производительность на любой платформе без изменения кода.

### 5.2 Допущения и крайние случаи
Допущение	Обработка
dim == 0	Возврат 0.0 из EvaluateFunction
num_points == 0	Валидация возвращает false
lower_bounds[j] >= upper_bounds[j]	Валидация возвращает false
num_threads == 0 (hardware_concurrency)	Установка 2 или 1
num_points < num_threads	Ограничение числа потоков до num_points
Нет TBB/OpenMP в системе	#ifdef отключает, ALL переключается на STL/SEQ

### 5.3 Использование памяти
Плоское представление: все данные хранятся в std::vector<double> (lower_bounds, upper_bounds)
Локальные буферы: std::vector<double> point(dim) создаётся в каждом потоке для избежания гонок данных
Оптимизация TBB: буфер создаётся внутри лямбды — один раз на блок, не на точку
STL: std::future хранит результаты каждого потока до завершения
Общая память: N × dim × sizeof(double) для точек + служебные структуры потоков
Рекомендация: при N > 10^7 использовать прогрессивный вывод или ограничить размер блока
